Exercício 04.10.2022 - Padrões Web - Patrícia Gomes

1) Responda as perguntas abaixo:

a) Na tag <input> com atributo type="text", para que serve o atributo "placeholder"?

O atributo placeholder especifica uma dica curta que descreve o valor esperado de um campo de entrada (por exemplo, um valor de amostra ou uma breve descrição do formato esperado). A dica curta é exibida no campo de entrada antes que o usuário insira um valor. Observação: o atributo placeholder funciona com os seguintes tipos de entrada: texto, pesquisa, url, tel, email e senha. Sua sintaxe é: < input placeholder="text">.

b) Na tag < input> com atributo type="radio", para que serve o atributo "checked"?

O atributo checked retorna o estado marcado de um botão de opção. Esta propriedade reflete o atributo HTML verificado. Sua sintaxe é: radioObject.checked = verdadeiro -> o Botão radio está marcado radioObject.checked = falso -> o Botão radio não está marcado.

c) Na tag <input> para que serve o atributo "value"?

O atributo value especifica o valor de um elemento <input >. O atributo value é usado de forma diferente para diferentes tipos de entrada:

  • Para "botão", "redefinir" e "enviar" - define o texto no botão;
  • Para "texto", "senha" e "oculto" - define o valor inicial (padrão) do campo de entrada;
  • Para "checkbox", "radio", "image" - define o valor associado à entrada (este também é o valor que é enviado no envio);

PS: O atributo value não pode ser usado com <input type="file">.

d)Qual atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

O atributo é o checkbox e a sintaxe é: <input type="checkbox">. É ele que define uma caixa de seleção. A caixa de seleção é mostrada como uma caixa quadrada que é marcada quando ativada. As caixas de seleção são usadas para permitir que um usuário selecione uma ou mais opções de um número limitado de opções. É sempre bom adicionar a tag <label> para melhores práticas de acessibilidade.

e) Para que serve a tag <label>? Para que serve o atributo "for" nessa tag?

A tag <label> define um rótulo para vários elementos, como por exemplo:

  • <input type="checkbox">
  • <input type="color">
  • <input type="data">
  • <input type="datetime-local">
  • <input type="email">
  • <input type="arquivo">
  • <input type="mês">
  • <input type="número">
  • <input type="senha">
  • <input type="radio">
  • <input type="intervalo">
  • <input type="pesquisa">
  • <input type="tel">
  • <input type="texto">
  • <input type="time">
  • <input type="url">
  • <input type="semana">
  • <metro>
  • <progresso>
  • <selecione>
  • <textarea>

O uso adequado de rótulos com os elementos acima beneficiará usuários de leitores de tela (lerão em voz alta o rótulo, quando o usuário estiver focado no elemento).

Usuários que têm dificuldade em clicar em regiões muito pequenas (como caixas de seleção) porque quando um usuário clica no texto dentro do elemento <label>, ele alterna a entrada e isso aumenta a área de acerto.

O atributo "for" de <label> deve ser igual ao atributo "id" do elemento relacionado para vinculá-los.

Um rótulo também pode ser vinculado a um elemento colocando o elemento dentro do elemento <label>.

f) Na tag <option> para que serve o atributo "selected"?

O atributo selected é um atributo booleano. Quando presente, especifica que uma opção deve ser pré-selecionada quando a página for carregada. A opção pré-selecionada será exibida primeiro na lista suspensa.

PS: O atributo selected também pode ser definido após o carregamento da página, com um JavaScript.

g) Nas tags de formulário para que serve o atributo "required"?

O atributo required é um atributo booleano. Quando presente, especifica que um campo de entrada deve ser preenchido antes de enviar o formulário. Observação: o atributo required funciona com os seguintes tipos de entrada: texto, pesquisa, url, telefone, e-mail, senha, seletores de data, número, caixa de seleção, rádio e arquivo.

h) Na tag <form> para que serve o atributo "action"?

O atributo action especifica para onde enviar os dados do formulário quando este é enviado através da internet.

Sua sintaxe é: <form action="URL">.

URL = Destino dos dados do formulário. Valores Possíveis:
Um URL absoluto - aponta para outro site (como action = "http://www.example.com/example.htm");
Um URL relativo - aponta para um arquivo dentro de um site (como action="example.htm")

i) Na tag <form> para que serve o atributo "method"?

O atributo method especifica como enviar dados de formulário (os dados de formulário são enviados para a página especificada no atributo action). Os dados do formulário podem ser enviados como variáveis de URL com o atributo (method="get") ou como transação de postagem HTTP com o atributo (method="post").

Observações sobre o método GET:

  • Anexa dados de formulário ao URL em pares de nome/valor;
  • O comprimento de um URL é limitado (cerca de 3.000 caracteres);
  • Nunca use GET para enviar dados confidenciais! (será visível no URL);
  • Útil para envios de formulários em que um usuário deseja marcar o resultado;
  • GET é melhor para dados não seguros, como strings de consulta no Google.

Observações sobre o POST:

  • Anexa dados de formulário dentro do corpo da solicitação HTTP (os dados não são mostrados na URL);
  • Não tem limitações de tamanho;
  • Envios de formulários com POST não podem ser marcados.

j) Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag <form>?

O valor a ser utilizado do atributo method é o “post”, uma vez que ele não mostra os dados na URL.

k) Qual valor deve ser informado no atributo "method" na tag <form> em um formulário que será usado para fazer upload de arquivos?

O valor a ser utilizado do atributo method é o “post”, uma vez que ele não possui limitação de tamanho dos dados.
